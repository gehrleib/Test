## Building an Executable with PyInstaller

You can package `CUBE.py` into a single executable file using PyInstaller. This allows the script to be run on systems without a Python interpreter, provided all dependencies are bundled.

### Prerequisites for Building
* Python 3.7+ (the same environment used to develop/run `CUBE.py`)
* PyInstaller: Install it via pip if you haven't already:
    ```bash
    pip install pyinstaller
    ```
* All Python package dependencies for `CUBE.py` must be installed in your Python environment (e.g., `aiohttp`, `orjson`, `diff-match-patch`, `python-dotenv`, `xmltodict`).

### Steps to Build

1.  **Navigate to the Script Directory:**
    Open your terminal or command prompt and change to the directory where `CUBE.py` is located.

2.  **Run PyInstaller:**
    Execute the following command in your terminal:
    ```bash
    pyinstaller --onefile --name CUBE CUBE.py
    ```
    * `--onefile`: This option bundles everything your script needs into a single executable file.
    * `--name CUBE`: This sets the name of your output executable to `CUBE.exe` on Windows, or `CUBE` on macOS/Linux.
    * `CUBE.py`: This is the main Python script you are packaging.

3.  **Locate the Executable:**
    After PyInstaller finishes, you will find a `dist` subfolder in your script's directory. Inside `dist`, you'll find the generated executable (e.g., `dist/CUBE.exe`).

### How to Run the Compiled Executable

1.  Copy the executable file (e.g., `CUBE.exe`) from the `dist` folder to your desired location.
2.  **Important:** Create or place your `config.txt` file in the **same directory** as the `CUBE.exe` executable. The script `CUBE.py` is designed to look for `config.txt` in the directory where the executable is running. The `config.txt` file is **not** included inside the executable with the command above.
3.  Run the executable (e.g., by double-clicking `CUBE.exe` on Windows or running `./CUBE` from the terminal on macOS/Linux).
4.  Output files generated by the script will be placed in an `output` subdirectory, also located in the same directory as the executable.

### Potential Build Issues & Tips
* **Missing Modules (`ModuleNotFoundError`):** If the executable fails at runtime due to missing modules (especially ones that are imported dynamically), you may need to use the `--hidden-import MODULENAME` option with PyInstaller. For example:
    ```bash
    pyinstaller --onefile --name CUBE --hidden-import some_module --hidden-import another_module.submodule CUBE.py
    ```
* **Data Files for Dependencies:** Some libraries might require their own data files. PyInstaller often has "hooks" to include these automatically. If not, you might need the `--add-data` option for those specific library files.
* **Clean Build:** If you re-run PyInstaller, it's a good idea to delete the `build` and `dist` folders first to ensure a clean build process.
* **Console Window:** The generated executable will open a console window by default, which is appropriate for this script as it uses logging.
