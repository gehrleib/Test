Currently, segment assignments in RegComp are manual and applied at the regulator level. This presents a challenge, as any change to a regulator's segment impacts all associated citations. To address this, we propose moving segment assignment from the regulator to the issuance level within RegComp. This shift will provide more granular control over which sections are affected by segment changes and enables segments to be automatically populated from CUBE based on completed RegFlow/RegInsight data. For even finer control, users will have the option to override the segment for individual citations by enabling an editing mode for the citation's segment field.
